# Аналитическая экосистема социальной сети 

## 📝 Описание проекта
В этом репозитории представлен комплексный проект по продуктовой, сквозной аналитике и Data Science для социальной сети. Основной функционал платформы включает просмотр ленты постов, оценку контента (лайки) и обмен личными сообщениями.

**Глобальная цель проекта:** Обеспечение технической стабильности сервиса, оптимизация затрат на инфраструктуру, рост вовлеченности на основе данных.

---

## 🏗️ Архитектура и направления аналитики

Проект декомпозирован на 4 специализированных аналитических блока:

### 1. Мониторинг "здоровья" платформы (`/dashboards`)
*   **Цель:** Разработка автоматизированных BI-решений для оперативного контроля технических и продуктовых метрик в реальном времени.
*   **Ключевые метрики:** DAU/MAU,  число просмотров и лайков, CTR, сегментация пользователей.
*   **Стек технологий:** SQL, Redash, Superset.

### 2. Продуктовая аналитика и поиск узких мест (`/product-analytics`)
*   **Цель:** Исследование падение DAU, сегментация пользователей и выявление проблемных групп.
*   **Стек технологий:** SQL, Redash, Superset.

### 3. A/B-тестирование системы рекомендаций (`/ab-testing`)
*   **Цель:** Статистическая оценка эффективности нового ML-алгоритма рекомендации постов для роста вовлеченности.
*   **Метрики эксперимента:** CTR постов.
*   **Методология:** Расчет размера выборки (MDE, мощность), Bootstrap, t-test / критерий Манна-Уитни, проверка на корректность разделения трафика.
*   **Стек технологий:** Python (SciPy, Statsmodels), Jupyter Notebooks.

### 4. Оценка маркетинговых эффектов и флешмобов (`/causal-inference`)
*   **Цель:** Оценка краткосрочного и долгосрочного влияния вирусных активностей (пользовательского флешмобов) на ключевые продуктовые метрики, когда проведение классического A/B-теста невозможно.
*   **Что анализируем:** Изменение объемов просмотренных постов, генерации постов и активности (DAU и CTR) во время и после проведения флешмоба. 
*   **Методология:** Причинно-следственный анализ (Causal Inference) с использованием байесовских моделей структурных временных рядов (Structural Time-Series).
*   **Стек технологий:** Python (`causalimpact` / `CausalImpact`), Orbit, ClickHouse.

### 5. Прогнозирование нагрузки на серверы (`/load-forecasting`)
*   **Цель:** Предсказание входящего трафика и нагрузки на инфраструктуру (RPS — Requests Per Second) для настройки автомасштабирования и предотвращения сбоев (downtime).
*   **Подход:** Анализ временных рядов с учетом суточной/недельной сезонности и аномальных всплесков (флешмоб).
*   **Стек технологий:** Python, Orbit, ClickHouse.

## 📊 Ключевые инсайты и результаты
*   **Продукт:** Наблюдается устойчивый рост аудитории по неделям, при этом растет как число новых пользователей, так и тех, кто остается в продукте, число ушедших пользователей растет линейно, при этом отток пользователей перекрывается привлечением новых клиентов в продукт, в целом наблюдается рост.
*   **A/B-тест:** Вводить новый алгоритм рекомендации не рекомендуется, так как наблюдается статистически значимое падение медианного CTR на , то есть удержание среднестатистического пользователя снижается, а для пользователей, демонстрирующих аномально высокую вовлеченность нельзя гарантировать стаблильное увеличение CTR, однотипный контент может вызвать временный всплеск интереса, который угаснет через некоторое время.
*   **Оценка флешмоба:** Флэшмоб во время проведения увеличил число просмотром постов на 96%, но не повлиял статистически значимо на число новых постов, DAU и CTR, он также не имел долгосрочных эффектов, метрики вернулись к своим значениям после окончания.
